💡 TIPS & BEST PRACTICES
═══════════════════════════════════════════════════════════════════

Development:
  • Changes in src/ reload instantly (hot reload)
  • Use Supabase Studio to manage data visually
  • Check MailHog for emails sent by the app
  • Use Adminer for quick database queries

Database:
  • Every table has tenant_id for multi-tenancy
  • RLS policies enforce tenant isolation
  • Use Supabase Studio for safe data management
  • Never bypass RLS for development

Multi-Tenant:
  • Create tenants in Supabase Studio
  • Assign users to tenants with roles
  • Each user sees only their tenant's data
  • RLS enforces this automatically

📋 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════

Problem: Can't connect to frontend
  → Check http://localhost:8080/ is open
  → Run: npm run dev (if stopped)

Problem: Database error
  → Check containers: docker-compose ps
  → Restart: docker-compose restart
  → View logs: docker logs schoolflow-pro-supabase-db-1

Problem: Hot reload not working
  → Clear: rm -r node_modules/.vite
  → Restart: npm run dev

Problem: RLS denies access
  → Check user role in Supabase Studio
  → Verify user has correct tenant_id
  → See docs for RLS policy details
